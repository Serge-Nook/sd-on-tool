# SD-ON Tool

Автономное приложение для **Steam Deck**, расширяющее возможности настройки
системы. Объединяет встроенные функции без сторонних магазинов расширений и
загрузки плагинов из сети:

- **Радио** — интернет-радиостанции через встроенный HTML5 `<audio>` (CEF Steam).
- **Темы** — 20 локальных CSS-тем, применяемых инъекцией стилей в интерфейс Steam.
- **Цвета экрана** — управление насыщенностью (аналог VibrantDeck).
- **О программе** — сведения о версии и разработчике.

После установки в **Меню быстрого доступа (Quick Access Menu, кнопка «…»)**
игрового режима появляется вкладка **«SD-ON Tool»**.

Проект — частичный аналог системы плагинов Decky Loader, но полностью
самодостаточное решение: **не требует** Decky Loader, магазинов плагинов и
загрузки контента из сети (кроме потокового радио).

## Структура репозитория

```
install.sh                    — GUI-диспетчер (zenity/kdialog, fallback — CLI)
scripts/
  installer.sh                — логика установки/удаления
  gen-themes.sh               — генератор 20 CSS-тем
  build-sh.sh                 — сборка самодостаточного SD-ON-Tool-Installer.sh
payload/                      — компоненты, копируемые при установке
  sd-on-tool.js               — инжектируемый скрипт UI и вся бизнес-логика
  settings.json               — настройки по умолчанию
  themes/                     — 20 CSS-файлов тем
  assets/icons, assets/fonts  — ресурсы
```

## Установка

На Steam Deck (режим рабочего стола) скачайте из релиза
`SD-ON-Tool-Installer.sh`, сделайте его исполняемым и запустите — откроется
окно с кнопками **«Установить»** / **«Удалить»**:

```bash
chmod +x SD-ON-Tool-Installer.sh
./SD-ON-Tool-Installer.sh
```

Это самодостаточный файл: все компоненты (`sd-on-tool.js`, 20 тем, `assets/`,
`settings.json`) встроены внутрь и распаковываются при запуске.

Из клона репозитория можно запустить установщик напрямую:

```bash
./install.sh            # графический диалог или CLI
# или:
bash scripts/installer.sh install
```

Собрать `.sh`-пакет самостоятельно:

```bash
bash scripts/build-sh.sh   # результат: dist/SD-ON-Tool-Installer.sh
```

Установщик:

1. Создаёт `~/.local/share/sd-on-tool/` и копирует туда `sd-on-tool.js`,
   `themes/`, `assets/`, `settings.json`.
2. Определяет путь к `steamui/index.html`
   (`~/.steam/steam/steamui/index.html` или `~/.local/share/Steam/steamui/index.html`).
3. Сохраняет резервную копию оригинального `index.html` в `backup/`.
4. Перед `</body>` добавляет строку подключения скрипта:
   `<script src="file:///home/deck/.local/share/sd-on-tool/sd-on-tool.js"></script>`.

После установки **перезапустите Steam / игровой режим**.

## Удаление

Кнопка **«Удалить»** в установщике или:

```bash
bash scripts/installer.sh uninstall
```

Восстанавливается оригинальный `index.html` из резервной копии и удаляется
директория `~/.local/share/sd-on-tool/`.

## Хранение настроек

Значения по умолчанию поставляются в `payload/settings.json`. В рантайме
(контекст инъекции CEF) состояние хранится в `localStorage` клиента Steam,
так как прямого доступа к файловой системе из внедрённого скрипта нет.

## Ограничения и проверка на устройстве

- Структура QAM Steam проприетарна и меняется между версиями SteamOS. Скрипт
  добавляет вкладку в контейнер QAM при его обнаружении, иначе показывает
  плавающую кнопку-лаунчер как запасной вариант. Точную интеграцию необходимо
  проверять на реальном Steam Deck.
- Аппаратная регулировка насыщенности (`xrandr` / DRM) недоступна из контекста
  инъекции: раздел «Цвета экрана» применяет CSS-фильтр `saturate()` как
  визуальное приближение и передаёт запрос внешнему системному помощнику при
  его наличии (`window.SDONTool.setSaturation`).
